## 1. 핵심 개념

### 비즈니스 소설 전략
- **접근법**: 판타지/소설로 이상향을 먼저 그린다 → 역산으로 현실 비즈니스를 매칭한다
- **레퍼런스**: 아마존 "Press Release First" 방법론과 동일한 구조
- **차별점**: 숫자가 아닌 세계관으로 투자자/파트너를 설득

### 알렉산드리아 = 프로토콜 (장소가 아님)
- 팔공산 고정 건물이 아니라 **이식 가능한 프로그램**
- 장소는 변수 → 시즌제/계약 단위로 이동 가능
- 장소가 알렉산드리아를 유치하기 위해 **역입찰**하는 구조
- 부동산 리스크 제로

```
알렉산드리아 본사 (IP/콘텐츠/프로토콜 보유)
        ↓ 입찰/계약
  시즌 1: 팔공산 템플스테이
  시즌 2: 제주 게스트하우스
  시즌 3: 강원도 폐교
  시즌 N: 해외 리트릿 센터
```

## 2. 개인 드라이버 (미해결 질문들)

> 이 프로젝트의 원천. 학위가 아니라 상처로 얻은 권위.

| 경험 | 미해결 질문 |
|------|-------------|
| 작은누나의 조현병 | 정신의 붕괴와 영혼은 다른가 |
| 어머니의 치매 | 자아가 사라져도 그 사람은 남아있는가 |
| 기독교 성령/은사/통공 | 신이 실제로 인간에게 접속하는가 |
| 작은아버지 주지승 | 깨달음은 제도 안에서 가능한가 |
| 개신교 목사 지망 + 천주교 세례 + 성가대 | 나는 어느 집에도 못 들어간 사람인가 |
| 성가와 영성 | 소리가 신에게 닿을 수 있는가 |
| 매직 머쉬룸 (동굴) | 의식의 경계는 어디까지인가 |
| 요양원 마지막 게이트 | 죽음을 잘 죽는다는 게 뭔가 |

**이 질문들은 전부 동일한 방향을 가리킨다: 의식, 자아, 죽음.**

## 3. 부루마블 게이트 구조 (서사 뼈대)

> 게임도 아니고, 책도 아니고, 영상도 아닌 새로운 플랫폼

```
START — 출생/세계 진입
  │
  ├─ GATE 1 │ 정신의 붕괴     — 누나의 조현병
  ├─ GATE 2 │ 자아의 소멸     — 어머니의 치매
  ├─ GATE 3 │ 신의 접속       — 성령/은사/통공
  ├─ GATE 4 │ 제도의 안과 밖  — 주지승 vs 목사 지망
  ├─ GATE 5 │ 소리와 영혼     — 성가/음악/영성
  ├─ GATE 6 │ 금기의 문       — 동굴과 매직 머쉬룸
  ├─ GATE 7 │ 마지막 게이트   — 알렉산드리아 입성
  │
END — 죽음이 아니라 완성
```

### 게이트 → 실물 서비스 역산

| 게이트 | 실제 프로그램/서비스 |
|--------|---------------------|
| GATE 1, 2 | 치매/정신질환 가족 케어 프로그램 |
| GATE 3, 4 | 크로스 종교 대담 (유튜브 채널) |
| GATE 5 | 음악 영성 워크샵 |
| GATE 6 | 홀로트로픽 호흡 / 감각차단 리트릿 |
| GATE 7 | 입주 멤버십 |

## 4. 플랫폼 정체

| 기존 미디어 | 특성 |
|-------------|------|
| 책 | 일방향, 고정 |
| 게임 | 오락, 가상 |
| 영상 | 소비, 수동 |
| **알렉산드리아** | **세계관 + 실물 체험 + 장소 변수 + 커뮤니티** |

**= IP 기반 팝업 성소**

독자/참여자는 자신이 몇 번 게이트에 있는지 인식한다.
이 인식이 생기는 순간 **콘텐츠가 커뮤니티가 된다.**

## 5. 소설 서사 구조 (1인칭 자전적)

```
1막 — 상처
   조현병 누나의 방 / 치매 어머니의 눈빛 / 성가대에서 느낀 공백
   "나는 신을 만나고 싶었는데 끝내 못 만났다"

2막 — 방랑
   목사를 포기하고 / 세례를 받고 / 삼촌 절에 앉아보고
   매직 머쉬룸 얘기를 처음 들은 날

3막 — 설계
   죽기 전에 한 번은 제대로 물어보고 싶다
   그 공간을 만들기로 결심 = 알렉산드리아

4막 — 현실화
   팔공산, 티벳 승려, 무속인, 유튜브
   나 같은 사람들을 위한 마지막 집
```

## 6. 콘텐츠 플라이휠 (제로 예산)

```
웹툰 (세계관 원천)
    ↓ 화면녹화 + 저작권 프리 종교음악
유튜브 (트래픽 유입)
    ↓ 링크
도메인 알렉산드리아 (콘텐츠 허브)
    ↓ 아카이브 / 강의 링크 / 재해석 콘텐츠
다시 웹툰 소재로
```

### 저작권 프리 종교음악 소스
- 그레고리안 성가: IMSLP, musopen.org
- 불교 범패, 독경: 퍼블릭 도메인
- 티벳 불교 음악: 퍼블릭 도메인

### 유튜브 채널 기획
- 크로스 종교 대담: 티벳 승려 + 무속인 + 신부 + 목사 + 철학자
- 게이트 챌린지 시리즈
- 입주자 인터뷰 (동의 기반)

## 7. 웹툰 제작 파이프라인

### 기술 스택
- **소재**: 저작권 프리 이미지 + AI 생성물 합성 (GPU 불필요)
- **유닛화**: 캐릭터 눈/코/입/팔다리 분리 → 조합 시스템
- **자동화**: Python 스크립트로 패널 합성 및 스토리라인 생성
- **인스트럭션**: Claude Code가 방향성 → 세부 인스트럭션 변환
- **배치 처리**: WSL 내 Claude가 일괄 처리
- **원격 입력**: 텔레그램 봇으로 핸드폰에서 소재/방향 전송

```
[핸드폰] 텔레그램 봇 → 소재/방향성 전송
    ↓
[Claude Code] 방향성 → 인스트럭션 생성
    ↓
[Python 자동화] 유닛 조합 → 패널 합성
    ↓
[배치 출력] 웹툰 패널 완성 → 화면녹화 → 유튜브
```

## 8. 타깃 & 포지셔닝

### 타깃
- 엘리트/귀족 성향
- 인생의 마지막을 남들과 다르게 설계하고 싶은 사람
- 죽음을 두려움이 아닌 완성으로 바라보는 사람

### 포지셔닝
> "나 같은 사람들을 위한 마지막 집"

일반 요양원과의 차별점:
- 영적 탐구가 프로그램에 내장됨
- 크로스 종교 — 어느 종교도 강요하지 않음
- 세계관이 있는 공간
- 입주자가 게이트를 직접 선택

## 10. 세계관 심화 (외부 평가에서 추출)

### 장르 포지셔닝
**Black Mirror + Hotel + Monastery**
- 미스터리한 규칙이 있는 공간
- 다양한 인물이 각자의 이야기를 들고 들어옴
- 영적 탐구와 공포가 공존

### 알렉산드리아의 정체 — 죽음의 공항
> "마지막 정거장. 들어왔다가 나간 사람이 거의 없다."

이 규칙 하나가 세계관을 호스피스 + 수도원 + 미스터리로 확장한다.
공간 자체가 **단방향 게이트**라는 암묵적 설정.

### 알렉산드리아의 이름이 가진 의미
- Alexandria = 고대 최대의 도서관
- 도서관 = 이야기 저장소
- 즉, Alexandria Sanctuary = **죽음 앞에서 꺼내는 이야기들의 아카이브**
- 이름 자체가 세계관을 설명한다

### 무한 주인공 모델 (스토리 공장 구조)
알렉산드리아는 특정 주인공의 이야기가 아니다.
공간이 주인공을 **무한히 생성**한다.

```
재벌 회장   → 동굴에서 자기가 버린 아들을 만남
가수        → 동굴에서 끝내 못 부른 마지막 노래를 부름
작가        → 동굴에서 자기가 쓰지 못한 책을 발견
스님        → 동굴에서 환속하고 싶었던 자신을 만남
살인자      → 동굴에서 피해자를 만남
```

누구든 들어올 수 있다.
각자 동굴에서 **자기 인생의 미완성**을 만난다.
이게 에피소드 구조의 핵심 엔진.

### 픽션인가 실물 프로젝트인가
**둘 다다.** 이게 이 프로젝트의 핵심 전략.

```
소설/웹툰 (세계관 선점)
    ↓
브랜드 (IP 확보)
    ↓
실제 공간 (시즌제 팝업 성소)
```

소설이 먼저 퍼지면 공간이 따라온다.
공간이 먼저 생기면 소설이 설명서가 된다.
순서는 상관없다. **둘이 서로를 증명**한다.
