§1 · Overview

Why Alexandria

이 레포의 원래 컨셉은 "신개념 요양원 실버타운"이었다. 그 자리에 영성 엔진이 들어간다. 요양원이 도서관이 되는 구체적 사건.

Alignment

  • Alexandria — 기원전 3세기 대도서관. 소실 후 수천 년 복원 시도. 본 엔진이 그 디지털 복원.
  • Sanctuary — 성소/피난처. 샤먼·미사·수피·아야와스카 전부 "성소"를 요구.
  • 7 Gate — 이미 서사가 깔려 있음. 본 백서 플러그 10개와 거의 1:1 매칭.
  • Directory — library/, sanctuary/, forum/, philosophy/ 전부 스펙대로 대기 중.

Ground Rule

사용자 = 박씨 1명. 외부 판매 없음. B급 상담/무당 소비를 대체하는 자기관리 OS. 출력은 레퍼런스이지 레시피가 아니다. 판단은 항상 박씨 것.

Weight Formula

박씨가 직접 던진 아키텍처 문장을 선형 코드로 구현:

def compute_weights(narrative, metadata, forced_gate=None): weights = {p.name: p.weight_default for p in PLUGS} # 1. 키워드 부스트 for plug in PLUGS: hits = count_keywords(narrative, plug.keywords_trigger) weights[plug.name] += hits * 0.15 # 2. 메타데이터 강제 if metadata.get("is_dream"): weights["freud"] += 0.3; weights["jung"] += 0.3 if metadata.get("is_family_event"): weights["family_systems"] += 0.4 if metadata.get("anniversary_within_30d"): weights["env_trigger"] += 0.5 # 3. Gate 강제 부스트 (UI 클릭) if forced_gate: for p in [p for p in PLUGS if p.gate_id == forced_gate]: weights[p.name] += 0.5 # 4. ParksyProfilePlug 항상 1.0 weights["parksy_profile"] = 1.0 return normalize(weights, lock=["parksy_profile"])

Success Criteria

  • 재현 정확도 ≥ 80% (2026-04-24 로그 기준)
  • 추론 지연 ≤ 10초 (로컬 CPU)
  • 플러그 활성 다양성 ≥ 3개 이상 가중치 0.15 초과
  • Phase 1 비용 ≤ $2
  • 월 사용 ≥ 3회 (자동화 원칙 유지)

Failure Declaration

  • Phase 1 후 4주 내 월 사용 < 3회 → 디프리케이트
  • 재현 정확도 < 50% → v2 재설계
